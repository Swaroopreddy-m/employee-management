# Render Deployment & Online Verification Guide

This guide provides step-by-step instructions to deploy the migrated Spring Boot application and its PostgreSQL database onto Render, followed by steps to verify the online endpoints.

---

## Step 1: Deploy a PostgreSQL Database on Render

1. Log into your [Render Dashboard](https://dashboard.render.com).
2. Click **New +** and select **PostgreSQL**.
3. Configure the database details:
   - **Name**: `employee-db`
   - **Database Name**: `employee_db`
   - **User**: (Auto-generated or choose one)
   - **Region**: Choose the same region you will use for the web service (e.g., Oregon US-West or Frankfurt EU).
4. Click **Create Database**.
5. Once active, note the **Internal Database URL** (for Render services) and **External Database URL** (for local administration/verification).

---

## Step 2: Initialize Database Schema and Seed Data

To populate the Render database with the initial tables and sample data:
1. Connect to the Render database using any PostgreSQL client (e.g. `psql` or DBeaver) using the **External Database URL**.
2. Run the script found in [schema.sql](file:///c:/Users/user/Documents/Projects/employee-management/schema.sql) to create the tables, indices, and seed data.

---

## Step 3: Deploy the Spring Boot Web Service on Render

Render builds Java projects using Maven out-of-the-box.

1. Push your updated codebase (with the PostgreSQL dependencies and properties) to a repository on **GitHub** or **GitLab**.
2. On Render Dashboard, click **New +** and select **Web Service**.
3. Connect your repository.
4. Configure the Web Service settings:
   - **Name**: `employee-management-service`
   - **Language**: `Java` (or Docker if deploying via container)
   - **Build Command**: `./mvnw clean package -DskipTests`
   - **Start Command**: `java -jar target/employee-0.0.1-SNAPSHOT.war` (or `.jar` if configured as jar)
5. Scroll down to **Environment Variables** and add the following:
   - `PORT`: `8080` (Render will automatically route external traffic to this port)
   - `SPRING_DATASOURCE_URL`: `jdbc:postgresql://<RENDER_INTERNAL_DB_HOST>:<PORT>/employee_db` (Copy this from your Render PostgreSQL Internal Connection String)
   - `SPRING_DATASOURCE_USERNAME`: `<RENDER_DB_USER>`
   - `SPRING_DATASOURCE_PASSWORD`: `<RENDER_DB_PASSWORD>`
6. Click **Create Web Service**.
7. Wait for the build and deployment logs to show "Your service is live".

---

## Step 4: Online API Verification

Once deployed, verify all endpoints using your service's public URL (e.g., `https://employee-management-service.onrender.com`):

### 1. Database-Backed Authentication
```bash
curl -X POST https://employee-management-service.onrender.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"userId": "US001", "password": "password123"}'
```
- **Expected response**: `{"status": "LOGIN_SUCCESS"}`

### 2. Employee CRUD (HTTP Basic Authentication)
```bash
curl -X GET https://employee-management-service.onrender.com/api/employees \
  -u admin:admin123
```
- **Expected response**: JSON array of employees.

### 3. User Master Details
```bash
curl -X GET https://employee-management-service.onrender.com/api/users
```
- **Expected response**: JSON array of users.

### 4. Dynamic Page Builder Rendering
1. Open a browser and visit:
   - Capture Page: `https://employee-management-service.onrender.com/page/capture`
   - Confirm Page: `https://employee-management-service.onrender.com/page/confirm`
   - Acknowledge Page: `https://employee-management-service.onrender.com/page/acknowledge`
2. Verify that the correct inputs and layouts render dynamically based on the configuration.
